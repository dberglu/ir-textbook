## Data Generation

The `glossary-data.json` is generated by the Python script at the project root:

```bash
python3 generate-glossary-data.py
```

This script:
1. Parses `docs/glossary.md` to extract term definitions
2. Loads `docs/learning-graph/learning-graph.json` for taxonomy data
3. Loads `docs/learning-graph/color-config.json` for category colors
4. Maps terms to categories
5. Determines difficulty levels based on concept dependencies
6. Outputs structured JSON

## Integration with MkDocs

The glossary is embedded in MkDocs via `docs/glossary-app.md`:

```markdown
<iframe src="glossary-app/index.html" width="100%" height="900px" ...></iframe>
```

This allows the interactive app to work within the static MkDocs site while maintaining its Vue.js reactivity.

3. Test locally: `mkdocs serve`
4. View at: `http://localhost:8000/glossary-app/`
